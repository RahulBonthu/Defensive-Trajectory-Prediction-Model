<task type="auto">
  <name>Task 1: Implement scripts/run_training.py</name>
  <files>scripts/run_training.py</files>
  <action>
Create `scripts/run_training.py`. This is the orchestration script that:

1. Sets MPS fallback env var before any torch import:
   `import os; os.environ.setdefault("PYTORCH_ENABLE_MPS_FALLBACK", "1")`

2. Loads data (cleaned.parquet + splits.json), builds samples via build_samples().
   Print sample counts: train / val / test.

3. For each variant in ["A", "B"]:
   For each seed in [42, 123, 456]:
   - Instantiate train/val/test DataLoaders for the correct input_dim
   - Call wandb.init(project="defensive-trajectory-prediction", name=f"model_{variant}_seed{seed}", config=config_dict, reinit=True)
   - Call train_one_model(model_variant=variant, train_loader, val_loader, num_epochs=50, lr=1e-3, checkpoint_dir="models", seed=seed, wandb_run=run)
   - Call run.finish()
   - Collect per-seed test RMSE by loading the saved checkpoint and calling collect_per_play_rmse
   - Call save_per_play_csv for the seed
   - Append numpy array of rmse values to rmse_{variant.lower()}_seeds list

4. After all seeds for both variants:
   - Determine canonical best checkpoint (lowest best_val_loss across seeds for each variant) and copy to models/model_a_best.pt and models/model_b_best.pt using shutil.copy
   - Call build_ablation_table(rmse_a_seeds, rmse_b_seeds) — this writes results/ablation_table.csv
   - Compute and print per-position RMSE for both Model A and B using the canonical test loader

5. Print a final summary including: best val loss per variant, ablation table contents.

Use `if __name__ == "__main__": main()` pattern so the script can also be imported.

Implementation notes:
- Load the checkpoint for evaluation using: `model.load_state_dict(torch.load(ckpt_path, map_location=device)); model.eval()`
- The test DataLoader must use shuffle=False and the same num_workers for both A and B — ensures aligned per-play vectors for the paired Wilcoxon test
- After DefensiveTrajectoryDataset construction, the context_df is released internally (the class sets self.context_df=None); but in this script you can del the cleaned df after all datasets are constructed to free memory
- batch_size=256; drop_last=False on val/test loaders
